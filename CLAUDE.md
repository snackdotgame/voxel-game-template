# Agent Guidelines for Minion.Game Projects

This is a Minion.Game project generated by the Minion CLI. Package scripts are the source of truth
for local development, checks, builds, and publishing.

## Project Shape

- `minion.json` configures game entrypoints, assets, and public game metadata.
- `src/client.ts` is browser code loaded by Vite through the Minion host shell.
- `src/server.ts` is server code loaded by the restricted Minion runtime.
- `src/shared/*` is for types, constants, and helpers shared by client and server code.
- `.minion/types/*` contains generated ambient declarations for `minion:client` and
  `minion:server`.

## Local Development

- Use the project's package manager scripts. Do not add Minion-owned install, typecheck, lint, or
  format commands.
- `dev` runs the Vite client dev server and `minion dev` server runtime together through Turbo.
- Always open and test the local game through the Minion host shell at
  `http://127.0.0.1:3030/`.
- Do NOT use the Vite-only client port `3031` as the game URL for networked play; it does not
  provide the required Minion launch envelope.
- Local ports can be configured with `MINION_DEV_PORT`, `MINION_CLIENT_PORT`, and
  `MINION_CLIENT_HOST`.
- Run `check` before committing gameplay changes and `build` before publishing.

## Runtime Contract

- Client code imports `client` from `minion:client`.
- Server code imports `server` from `minion:server` and exports `main()`.
- Use `recv()` and `drain()` on datagram or stream channels.
- Do not use callback-style receive handlers or raw WebTransport/postMessage plumbing.
- Keep authoritative multiplayer state in server code. Treat browser state as display/input state.
- Validate network messages before trusting them.
- The server runtime is restricted. Do not use filesystem, process, native addons, raw sockets,
  server listeners, or outbound `fetch` unless Minion explicitly supports that capability.

## Publishing

- `publish` is the creator-facing command for building, uploading, and activating a game version.
- `deploy` is lower-level and mainly useful for debugging or CI upload flows.
- When asked to publish, deploy, release, or prepare the game for hosting, use the generated
  `minion-game-publish` skill before running commands.
- Use `minion auth whoami` to check authentication and `minion auth login` when credentials are
  missing.
- Configure public game metadata in `minion.json`, including `game.maxPlayers` and optional
  `game.titleImage`.
